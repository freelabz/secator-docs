# Glossary

A short cheat-sheet of platform-specific terms. Other sections link to these definitions.

| Term | Definition |
|---|---|
| <a id="g-organization"></a>**Organization** | The top-level tenant. Holds users, billing, and one or many workspaces. Switch organizations from the top-bar selector. |
| <a id="g-workspace"></a>**Workspace** | An engagement-scoped container: targets, runners, findings, reports, members, scope. Most actions in the UI are workspace-scoped. |
| <a id="g-target"></a>**Target** | An in-scope asset (URL, domain, subdomain, host, IP, CIDR). Belongs to a workspace; cannot be scanned without a legal-authorization confirmation. |
| <a id="g-task"></a>**Task** | The smallest unit of automation: one execution of one tool (`nmap`, `httpx`, `nuclei`, …) with its options. |
| <a id="g-workflow"></a>**Workflow** | A predefined chain of tasks that share inputs/outputs (e.g. `url_crawl` runs `katana → httpx → trufflehog`). 15 workflows ship with the platform. |
| <a id="g-scan"></a>**Scan** | The highest-level composition: a bundle of workflows for an entire engagement type (`domain`, `host`, `subdomain`, `url`, `network`). 5 scans ship with the platform. |
| <a id="g-runner"></a>**Runner** | The generic name for any executed task / workflow / scan. The *Runners* page lists all of them; *Tasks* / *Workflows* / *Scans* pages list only their own type. |
| <a id="g-chore"></a>**Chore** | A scheduled, recurring runner (attack-surface monitoring, periodic scans, cleanup). Created and toggled on the *Features* page (owner only). |
| <a id="g-finding"></a>**Finding** | A structured result produced by a runner: a vulnerability, a discovered URL, an open port, a leaked email, etc. Each finding has a type, a source tool, evidence, and a status. |
| <a id="g-exploit"></a>**Exploit** | A finding type representing a public PoC matched against a detected product/version. Sources: ExploitDB, Metasploit, GitHub. |
| <a id="g-record"></a>**Record** | A finding type holding a stored HTTP request/response pair, used as evidence on vulnerability findings. |
| <a id="g-tag"></a>**Tag** | A semantic marker on a finding (`email_address`, `aws_key`, `pattern:sqli`, `juicy_extension`). Generated by tooling like `cariddi`, `gf`, `trufflehog`, `gitleaks`. |
| <a id="g-suggestion"></a>**Suggestion** | An AI-generated, actionable hint based on existing findings. Either proposes running a specific runner, or adding a target the system identified. See [AI suggestions](ai-suggestions.md). |
| <a id="g-profile"></a>**Profile** | A preset of options applied in bulk to a runner — `aggressive`, `passive`, `stealth`, `default`. Picked on the *Run* form. |
| <a id="g-scope"></a>**Scope** | The allowlist + denylist of patterns that constrain what can be added as a target and what is included in suggestions/scans. Configured in workspace settings. |
| <a id="g-run-hours"></a>**Run hours** | The metering unit of platform usage. Visible in the user menu; capped per organization plan. Aggressive profiles consume faster than passive ones. |
| <a id="g-report"></a>**Report** | A client-ready deliverable assembled from selected findings of a workspace. Built in the Report editor; exportable. |
| <a id="g-token"></a>**API token** | A credential to call the Secator API outside the UI. Created in the *API Keys* page; the secret is shown once — copy it immediately. |
